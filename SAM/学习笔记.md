# 📚 Segment Anything Model 学习笔记


### SAM的目标：为图像分割构建一个基础模型
 promptable model，在大数据集上进行pre-trained，可用于下游任务微调。

### SAM项目的三大组件：task,model,data
- task: 什么任务适用zero shot
- model:用什么模型
- data: 有史以来最大的图像分割数据集

### Task
- 收到NLP的启发，在图像分割领域引入提示。
- prompt的设计：点，框，掩码
- 对于模糊的prompt，输出至少一个合理的掩码。


### Model
- 模型的三个主要组件: image encoder, prompt encoder, mask decoder
- image encoder：基于MAE的VIT(vision transformer)预训练模型
  用于将图像转化成模型可操作的embedding(嵌入)
- prompt encoder: 对提示输入的编码。prompt input 可分为两种，稀疏提示(point,box,text),
  稠密提示(mask)
- mask decoder:解码出分割好的掩码，基于transformer的解码器,分为四个step
1. token self-attention
2. 从token到image embedding的cross-attention
3. 每个token(point-wise)通过MLP进行特征增强
4. 从图像embedding到token的cross-attention
  以上第三步，第四步的计算结果继续放入encoder计算
1. 上采样：通过两个转置卷积层将更新后的图像嵌入上采样4倍注意力计算：token和图像Embedding再做一次自注意力计算，更新Embedding
2. MLP：将更新后Embedding传递给一个小型的3层MLP，输出一个与第一步上采样维度相匹配的向量预测： 
3. 通过空间逐点乘法，最终预测生成mask

以上decoder输出分割的预测结果和损失数值
- 分割结果默认给三个掩码，三个掩码的关系是从整体到局部。
    一般来说，当有多个输入的prompt时，输出越准确。
- Loss的计算，分为Mask Loss和IOU Loss，两者缩放相加得到最终结果

### Data Engine
- 数据引擎的构建分为三个阶段，逐渐从人工走向自动化
- 人工辅助阶段：由人工手动标注图像中的掩码
- 半自动化阶段：模型先标注出明显的mask，再由人工标注其他模型没有发现的mask，这一阶段，可以提升模型的准确性
- 全自动化阶段：让模型自动预测mask，通过一系列算法，最终筛选出合理的masks

- SAM的数据集不仅体量最大，而且在RAI(公平分析)上也比传统的数据集更加优秀

### 零样本实验
- 在多种类型的零样本实验中，SAM都展现出了更加优秀的性能

### 不足
- SAM的实时性分析：在image encoder阶段，每张图片的编码时间在0.1s-0.5s,延迟严重，主要是由于VIT架构。
在decoder阶段可以做到亚实时，这是因为iamge embedding是主要计算量，之后的解码任务速度较快。
- 可以考虑MobleSAM的变体，思路是用更小的编码backbone替换原始ViT
